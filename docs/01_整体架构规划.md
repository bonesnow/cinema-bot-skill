# Cinema Bot 整体架构规划

## 一、最终目标

用户在飞书中发送：

```text
我要看 星际穿越
```

系统自动完成：

```text
飞书消息
  ↓
身份白名单校验
  ↓
解析片名与画质偏好
  ↓
搜索自己的夸克网盘
  ├─ 已有：回复最佳现有版本，流程结束
  └─ 没有：搜索已配置的授权资源源
                 ↓
              汇总、过滤、去重
                 ↓
              画质评分与排序
                 ↓
              转存评分最高的版本
                 ↓
              飞书回复最终结果
```

## 二、模块边界

| 模块 | 目录/文件 | 职责 |
|---|---|---|
| Web 服务 | `app/main.py` | FastAPI 启动、健康检查、Webhook 路由 |
| 飞书通道 | `app/channels/feishu.py` | 验证事件、接收消息、发送文本/二维码 |
| 企业微信通道 | `app/channels/wecom.py` | 企业微信回调验证、解密、回复 |
| 业务编排 | `app/service.py` | 串联“先查网盘、再查资源、最后转存” |
| 夸克扫码 | `app/quark_auth.py` | 获取二维码、轮询扫码、持久化 Cookie |
| 夸克网盘 | `app/quark.py` | 搜索网盘、解析分享、保存到目标目录 |
| 资源源插件 | `app/providers/` | 本地目录或授权 JSON API 搜索 |
| 画质评分 | `app/scoring.py` | 4K、片源、HDR、音轨、编码、字幕评分 |
| 配置管理 | `app/config.py` | 读取环境变量、识别占位符、检查缺失项 |
| 配置检查 | `app/config_check.py` | 部署前输出必填项检查结果 |
| 数据持久化 | `./data:/data` | 保存扫码登录态和本地资源目录 |

## 三、完整处理规则

### 1. 消息入口

支持以下指令：

```text
我要看 片名
搜索 片名 4K
直接发送夸克分享链接
夸克登录
退出夸克
状态
帮助
```

### 2. 安全校验

飞书请求先校验 Verification Token，再进行事件去重和用户白名单判断。生产环境必须限制 `FEISHU_ALLOWED_OPEN_IDS`，避免其他人触发转存或覆盖夸克登录态。

### 3. 夸克优先

系统先用纯片名搜索自己的夸克网盘。即使用户输入“片名 4K HDR”，也会去掉画质词后搜索，避免因为文件命名差异漏检。

网盘命中多个结果时，按以下顺序选择最佳现有版本：

1. 分辨率；
2. 片源质量；
3. HDR / Dolby Vision；
4. 音轨；
5. 编码；
6. 字幕；
7. 文件大小与标题相似度。

### 4. 资源源聚合

只有网盘没有匹配内容时，才调用资源源。资源源必须是你自己维护或有权使用的 JSON API，或者本地合法资源目录。

统一数据结构：

```json
{
  "title": "影片名 2026 2160p WEB-DL HDR",
  "share_url": "https://pan.quark.cn/s/xxxx",
  "quality": "2160p WEB-DL HDR",
  "size": "25GB",
  "source": "authorized"
}
```

系统只接受夸克分享链接，不接受磁力或 BT 链接。

### 5. 自动选优

资源候选会先按分享链接去重，再按画质评分。默认优先级大致为：

```text
2160p / 4K > 1080p > 720p
BluRay / REMUX > WEB-DL > WEBRip > HDTV
Dolby Vision > HDR10+ > HDR10 > HDR
Atmos > TrueHD > DTS-HD > EAC3 > AAC
H.265 / HEVC > H.264
带中文字幕 > 无字幕标记
```

### 6. 转存与回执

- `DRY_RUN=true`：只显示最佳候选，不实际转存；
- `DRY_RUN=false` 且 `AUTO_SAVE=true`：自动转存最佳候选；
- 转存成功后回复片名、画质、大小、来源和结果；
- 失败时返回不含 Cookie 和密钥的错误摘要。

## 四、部署拓扑

推荐生产拓扑：

```text
Internet
   ↓ HTTPS 443
Caddy / Nginx
   ↓ HTTP 127.0.0.1:8000
Cinema Bot 容器
   ├─ /data/quark_auth.json
   ├─ /data/catalog.json
   ├─ 飞书开放平台 API
   ├─ 夸克网页接口
   └─ 你的授权资源 API
```

## 五、你需要负责的外部条件

> 下列项目无法由代码自动替你创建，是部署时需要提供的内容。

- 🔴 飞书企业自建应用；
- 🔴 飞书 App ID、App Secret、Verification Token；
- 🔴 一个可从公网访问的 HTTPS 域名和反向代理；
- 🔴 至少一个资源源：授权 API 或本地资源目录；
- 🔴 生产环境飞书用户白名单；
- 🟡 夸克目标目录 FID；不填则保存到根目录；
- 🟢 夸克 Cookie 无需填写，通过机器人扫码获取。

## 六、上线阶段

### 阶段 1：本地自检

- 填写 `.env`；
- 保持 `DRY_RUN=true`；
- 执行配置检查和测试；
- 启动容器并检查 `/healthz`。

### 阶段 2：飞书联调

- 配置回调地址；
- 发送“状态”；
- 发送“夸克登录”并扫码；
- 发送一个本地目录中的测试影片。

### 阶段 3：小范围试运行

- 配置自己的 open_id 白名单；
- 验证网盘已有时不搜索资源；
- 验证多个候选时选择最高画质；
- 检查目标目录是否正确。

### 阶段 4：正式转存

确认无误后修改：

```env
DRY_RUN=false
AUTO_SAVE=true
```
