# 05｜反向代理示例

把下面所有 `【需要你填写】` 替换为真实值。

## 方案 A：Caddy

```caddyfile
【需要你填写：bot.example.com】 {
    reverse_proxy 127.0.0.1:8000
}
```

飞书回调：

```text
https://【需要你填写：bot.example.com】/webhooks/feishu
```

## 方案 B：Nginx

```nginx
server {
    listen 443 ssl http2;
    server_name 【需要你填写：bot.example.com】;

    ssl_certificate     【需要你填写：证书完整路径】;
    ssl_certificate_key 【需要你填写：私钥完整路径】;

    client_max_body_size 10m;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
    }
}
```

建议同时配置 80 → 443 跳转。

## 防火墙

Docker 默认配置绑定：

```text
127.0.0.1:8000
```

因此外部访问只能通过反向代理。不要修改为 `0.0.0.0:8000`，除非你已经配置额外的防火墙和认证。
