# 04｜上线验收清单

## A. 配置检查

- [ ] `.env` 中不存在 `__FILL_`；
- [ ] `FEISHU_APP_ID` 已填写；
- [ ] `FEISHU_APP_SECRET` 已填写；
- [ ] `FEISHU_VERIFICATION_TOKEN` 已填写；
- [ ] 已配置授权 API 或 `data/catalog.json`；
- [ ] 正式环境已配置 `FEISHU_ALLOWED_OPEN_IDS`；
- [ ] `QUARK_COOKIE` 保持为空；
- [ ] 初次测试 `DRY_RUN=true`。

执行：

```bash
docker compose run --rm cinema-bot python -m app.config_check
```

- [ ] 输出没有“🔴 必须补充”。

## B. 服务检查

- [ ] `docker compose ps` 显示 healthy；
- [ ] `/healthz` 返回 `ok: true`；
- [ ] `/setup/status` 返回 `ready: true`；
- [ ] 公网 HTTPS 域名可以访问；
- [ ] 8000 端口未直接暴露公网。

## C. 飞书检查

- [ ] 回调地址验证成功；
- [ ] 发送“状态”能回复；
- [ ] 非白名单用户会被拒绝；
- [ ] 同一事件重复推送不会重复执行；
- [ ] 二维码图片能正常发送。

## D. 夸克检查

- [ ] 发送“夸克登录”可以扫码；
- [ ] 扫码成功后“状态”显示已登录；
- [ ] `/data/quark_auth.json` 已生成；
- [ ] 容器重启后登录态仍存在；
- [ ] `data/quark_auth.json` 没有提交到 Git；
- [ ] 目标目录 FID 已验证。

## E. 业务检查

- [ ] 网盘已有目标时，机器人直接回复且不调用资源 API；
- [ ] 网盘没有目标时，才调用资源 API；
- [ ] 多个资源结果能选择最高画质；
- [ ] 非夸克链接被忽略；
- [ ] `DRY_RUN=true` 时不会真实转存；
- [ ] 直接发送合法夸克链接能被识别；
- [ ] 错误回复不会泄露 Cookie 或 Token。

## F. 开启真实转存

以上全部通过后：

```env
DRY_RUN=false
AUTO_SAVE=true
```

重启并使用一个可删除的测试资源做最后验证：

```bash
docker compose up -d --build
```
