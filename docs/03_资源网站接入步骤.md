# 03｜资源网站接入步骤

本模块不内置任何具体资源站。最简单方式只需要输入你有权使用、允许自动化访问的网站网址；程序会用通用启发式适配器搜索页面、详情页和夸克分享链接。

## 一、推荐方式：只输入网址

```bash
./scripts/configure_sources.sh https://你的资源站.example
```

多个网站：

```bash
./scripts/configure_sources.sh https://site-a.example https://site-b.example
```

脚本会生成：

```text
data/websites.yaml
```

并自动写入：

```env
WEBSITE_CONFIG_PATH=data/websites.yaml
```

本地问答直接使用这个路径；VPS 一键部署会自动切换为容器里的 `/data/websites.yaml`。

只有自动识别失败时，才需要继续看下面的高级 CSS 配置。

## 二、高级模式：手动 CSS 配置

| 网站类型 | `mode` | 说明 |
|---|---|---|
| 搜索结果直接存在网页源代码中 | `html` | 推荐，速度快、资源占用低 |
| 必须执行 JavaScript 后才出现结果 | `browser` | 使用 Playwright Chromium |
| 有验证码、滑块或强制人工确认 | 不建议自动化 | 需要人工处理，程序不会绕过 |

## 三、复制模板

```bash
cp websites.example.yaml data/websites.yaml
```

打开 `data/websites.yaml`，搜索：

```text
__FILL_
```

选择一个模板填写，另一个保持 `enabled: false`。

## 四、必须填写的字段

```yaml
name: "🔴【网站名称】"
enabled: true
authorized: true
mode: "html"

allowed_domains:
  - "🔴【网站域名，不带路径】"

search:
  url: "🔴【搜索页 URL】"
  query_param: "🔴【搜索参数名】"

selectors:
  result_item: "🔴【每条结果 CSS】"
  title: "🔴【标题 CSS】"
```

如果夸克链接在详情页，还需要：

```yaml
selectors:
  detail_url: "🔴【详情页链接 CSS】"

detail:
  share_url_selector: "🔴【详情页夸克链接 CSS】"
```

## 五、如何找 CSS 选择器

1. 在电脑浏览器打开网站并搜索一个确定存在的内容；
2. 按 `F12` 打开开发者工具；
3. 使用元素选择工具点中一条结果；
4. 确定包住整条结果的元素，填写 `result_item`；
5. 在该结果内部找到标题元素，填写 `title`；
6. 找到详情页 `<a>` 元素，填写 `detail_url`；
7. 打开详情页，找到夸克链接元素，填写 `share_url_selector`。

选择器示例仅用于解释语法：

```yaml
result_item: ".search-results .result-card"
title: ".result-title"
detail_url: "a.detail-link"
share_url_selector: "a.quark-link"
```

不要照抄示例，必须以目标网站实际 HTML 为准。

## 五、需要登录的网站

在 `websites.yaml` 指定一个环境变量名：

```yaml
cookie_env: "SITE_HTML_COOKIE"
```

再在 `.env` 填写：

```env
SITE_HTML_COOKIE=🔴【网站允许你使用的登录 Cookie】
```

注意：

- Cookie 属于敏感凭证，不要提交到 Git；
- 网站 Cookie 与夸克 Cookie 无关；
- 机器人不会破解验证码、滑块或绕过登录限制；
- Cookie 失效后需要重新获取。

## 六、动态网站

动态网站使用：

```yaml
mode: "browser"
```

优先把关键词直接放到 URL：

```yaml
search:
  url: "https://example.com/search?keyword={query}"
```

如果必须操作搜索框：

```yaml
search:
  url: "https://example.com"
  input_selector: "input[name=q]"
  submit_selector: "button[type=submit]"
  result_wait_selector: ".result-card"
```

Docker 镜像已包含 Chromium。动态模式占用内存明显高于 HTML 模式，建议服务器至少预留 1GB 可用内存。

## 七、检查配置

```bash
docker compose run --rm cinema-bot python -m app.website_check
```

测试实际搜索，但不会转存：

```bash
docker compose run --rm cinema-bot python -m app.website_test "测试片名"
```

输出中会显示网站名称、标题、画质、大小和评分，不会打印网站 Cookie 或夸克 Cookie。

## 八、限制与维护

网站页面结构改变后，CSS 选择器可能失效。此时只需修改 `data/websites.yaml`，无需修改业务编排、画质评分或夸克转存代码。
