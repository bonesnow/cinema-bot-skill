# 12｜部署方式

这个项目按使用条件分成三种部署方式。先选模式，再看配置。

## A. 没有 VPS，也没有域名：本地问答

这是最低门槛的使用方式，不接飞书、不需要公网、不需要 HTTPS。

```bash
./scripts/local_qa.sh
```

脚本会自动准备：

- `.env`
- `data/catalog.json`
- Python 虚拟环境 `.venv`
- 项目依赖

启动后在终端里输入：

```text
状态
我要看 星际穿越
退出
```

这种模式默认使用本地目录：

```text
data/catalog.json
```

你需要把自己有权使用的夸克分享写进去。公开仓库里的 `catalog.example.json` 只是格式示例。

## B. 没有 VPS，但想在本机跑服务

适合本机调试 API、健康检查和 Webhook 逻辑：

```bash
./scripts/deploy_local.sh
```

启动地址：

```text
http://127.0.0.1:8000
```

健康检查：

```bash
curl http://127.0.0.1:8000/healthz
```

飞书要求公网 HTTPS 回调。本机服务如果没有公网域名，飞书无法直接调用；这时继续使用本地问答模式即可。

## C. 有 VPS：一键 Docker 部署

在 VPS 上执行：

```bash
./scripts/deploy_vps.sh
```

脚本会自动：

- 创建 `.env` 和 `data/catalog.json`
- 检查 Docker Compose
- 根据 `.env` 判断是否启用 Caddy
- 构建并启动服务
- 输出健康检查地址和飞书回调地址

如果你有域名，在 `.env` 填：

```env
PUBLIC_HOST=bot.example.com
```

然后重新执行：

```bash
./scripts/deploy_vps.sh
```

飞书回调：

```text
https://bot.example.com/webhooks/feishu
```

如果没有域名，VPS 脚本只启动本机绑定服务：

```text
http://127.0.0.1:8000
```

这时无法直接接飞书公网回调，除非你另行配置 HTTPS 域名、反向代理或临时隧道。

## 推荐顺序

1. 先用 `./scripts/local_qa.sh` 跑通问答和资源源。
2. 需要飞书时，再准备公网 HTTPS 域名。
3. 有 VPS 后使用 `./scripts/deploy_vps.sh` 长期运行。
