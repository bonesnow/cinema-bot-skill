# 02｜飞书接入步骤

## 一、需要从飞书后台取得的 3 个值

```env
# 🔴【需要你填写】
FEISHU_APP_ID=__FILL_FEISHU_APP_ID__

# 🔴【需要你填写】
FEISHU_APP_SECRET=__FILL_FEISHU_APP_SECRET__

# 🔴【需要你填写】
FEISHU_VERIFICATION_TOKEN=__FILL_FEISHU_VERIFICATION_TOKEN__
```

不要把 App Secret 或 Verification Token 发到群聊、截图公开或提交到 Git。

## 二、应用设置

1. 创建企业自建应用；
2. 开启机器人能力；
3. 配置机器人接收文本消息、发送消息和发送图片所需权限；
4. 发布可用版本并将应用加入测试范围；
5. 事件订阅地址填写：

```text
https://【需要你填写：你的域名】/webhooks/feishu
```

6. 订阅消息接收事件；
7. 完成回调地址验证。

## 三、回调联调

服务启动后执行：

```bash
curl http://127.0.0.1:8000/healthz
```

检查：

```json
{
  "ok": true,
  "configuration_ready": true,
  "configured_channels": ["feishu"]
}
```

然后在飞书给机器人发送：

```text
状态
```

收到回复后，再发送：

```text
夸克登录
```

机器人应依次发送：

1. 正在生成二维码；
2. 二维码图片；
3. 扫码时限提示；
4. 登录成功或失败结果。

## 四、设置用户白名单

生产环境填写：

```env
# 🔴【生产环境需要你填写】
FEISHU_ALLOWED_OPEN_IDS=ou_xxx,ou_yyy
```

多个 open_id 用英文逗号分隔，不要有中文逗号。

设置后重启：

```bash
docker compose up -d --build
```

## 五、常见定位顺序

### 回调验证失败

检查：

- 域名是否能从公网访问；
- HTTPS 证书是否正常；
- 路径是否为 `/webhooks/feishu`；
- `FEISHU_VERIFICATION_TOKEN` 是否完全一致；
- 反向代理是否保留 POST 请求体。

### 能收到消息但不能回复

检查：

- App ID / App Secret；
- 应用是否已发布；
- 机器人是否在可用范围；
- 是否拥有发送消息和图片权限；
- 容器日志：`docker compose logs -f cinema-bot`。

### 文本能回复但二维码不能发送

重点检查图片上传和图片消息相关权限。
