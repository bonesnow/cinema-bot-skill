# 11｜媒体库整理和分类刮削

## 这次补回了什么

原始 `cinema-manager` 的后处理能力包括：

1. 转存完成后获取已保存文件 FID；
2. 识别电影或电视剧；
3. 创建 Plex、Jellyfin、Infuse 兼容目录；
4. 对杂乱文件名进行标准化；
5. 对规范片源名保留原名；
6. 使用 OMDb 获取类型并按类型分类；
7. 缓存已获取的元数据。

本版本已经把上述流程接到飞书点片工作流后面。

## 完整流程

```text
飞书点片
  ↓
先查夸克网盘
  ↓
资源源搜索并选择最高画质
  ↓
转存到 QUARK_TARGET_FID
  ↓
轮询夸克转存任务并取得 saved_fids
  ↓
识别电影 / 剧集 / 年份 / 季集信息
  ↓
OMDb 分类（未配置则进入“其他”）
  ↓
创建目录、重命名、移动
  ↓
飞书返回最终媒体库路径
```

## 配置

运行：

```bash
./scripts/configure_library.sh
```

或手动编辑 `.env`：

```env
ORGANIZE_AFTER_SAVE=true
LIBRARY_ROOT_NAME=夸克影视
LIBRARY_ROOT_FID=
LIBRARY_GENRE_FOLDERS=true
LIBRARY_KEEP_SCENE_NAMES=true
OMDB_API_KEY=
METADATA_CACHE_PATH=/data/metadata_cache.json
```

### LIBRARY_ROOT_FID

- 留空：程序在夸克根目录自动创建 `LIBRARY_ROOT_NAME`；
- 已有媒体库目录：填写该目录 FID，程序直接使用它；
- 不要填写普通文件 FID。

### OMDB_API_KEY

- 留空：仍可标准化命名和整理目录，并使用标题、来源文本做基础分类；
- 填写：可辅助判断华语电影、外语电影、动画电影、国产剧、欧美剧、日韩剧、动漫、纪录片、综艺、儿童等分类；
- 缓存在 `/data/metadata_cache.json`，相同影片不会反复请求。

## 飞书指令

```text
我要看 星际穿越
```

搜索、转存并自动整理。

```text
整理 星际穿越
```

维护命令：整理网盘中已经存在的指定影片。

```text
整理网盘
```

维护命令：整理 `QUARK_TARGET_FID` 下最多 20 个顶层项目。该命令会移动和重命名文件，执行前确认目标目录设置正确。

```text
刮削 星际穿越 2014
```

维护命令：查看识别结果和 OMDb 分类信息。

```text
状态
```

查看自动整理、媒体库目录和刮削状态。

## 关于“刮削”的范围

当前刮削用于目录分类、年份、类型、简介和 IMDb ID 缓存。海报、背景图和 NFO 不会上传到夸克网盘；文件命名标准化后，由 Plex、Jellyfin 或 Infuse 自行完成丰富媒体墙刮削。
